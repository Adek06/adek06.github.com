如何控制坦克的行为

移动

最开始需要控制的先是坦克的移动。在TankMovement这个文件里：

 
x
    public float speed=10;
    public float angularSpeed = 10;
    public int number = 1;
​
    void Update(){
        float v = Input.GetAxis("VerticalPlayer"+number);
        rigidbody.velocity = transform.forward * v * speed;
      
        float h = Input.GetAxis("HorizontalPlayer"+number);
        rigidbody.angularVelocity = transform.up * h * angularSpeed;
    }

前三行设定坦克的速度、旋转速度，以及编号。编号主要是用来区分两个不同的坦克，这个最后再谈。之后在Update()函数里，第一行，让v来获取键盘输入，垂直前则正1，退则负1。"VerticalPlayer"是在按键设置里取得名字，也就是说，这个名字可以自己取，和内部函数无关。

之后让刚体的速度等于位置在前进这个方向 乘以 键盘的输入的正负性 再乘以 速度

后面的两行同上。

transform.forward代表着Z轴，transform.up大概就是Y轴吧，transform.right就是X轴。

还有一个地方是在Unity引擎里做的设置。Rigidbody里的两个选项Freeze(锁定) Position 、Freeze Rotation。

因为只需要控制坦克在X、Z轴的移动（在Unity里，Y轴指向上）所以，需要锁定Y轴，事实上，因为没有设定Y轴上的按键，这点不需要设置也没关系。然后就是锁定旋转，只绕Y轴旋转，因此就要勾选X、Z。

攻击

关于攻击。大概就是按键之后，在坦克的炮口的前方一小点的位置生成一个炮弹对象，之后控制炮弹的速度朝着坦克当前方向移动，加上刚体之后，子弹会自由落体。

于是，写下第一个脚本TankAttack.cs

 
x
    public KeyCode fireKey = KeyCode.Space;
    public GameObject shellPrefab;
    public float shellSpeed = 10;
    private Transform firePosition;
    void Start () {
        firePosition = this.GetComponent<Transform>();
    }
    void Update () {
        if (Input.GetKeyDown(fireKey))
        {
            GameObject go = GameObject.Instantiate(shellPrefab, firePosition.position, firePosition.rotation);
            go.GetComponent<Rigidbody>().velocity = go.transform.forward*shellSpeed;
​
        }
    }

第一行设置按键，当按下空格建（Space）之后得到一个非0值。第二行接收炮弹对象。之后设置炮弹的移动速度。“firePosition”则是炮弹出现的位置。

首先，在start()函数里，获取炮弹出现的位置

Update()函数里，判断，如果按下了空格键，则生成炮弹对象，在它该出现的位置和角度，而它的刚体的移动速度，等于超前的方向向量乘以速度。

到这里，坦克已经能做到攻击和移动了。

如何判断攻击中目标

攻击中目标的判断这里，交给炮弹来处理。首先炮弹需要有一个碰撞器。其次要把碰撞检测关了，避免穿过物体之后，发生第二次碰撞检测。

在Shell.cs里：

 
x
    void OnTriggerEnter(Collider collision)
    {
        if (collision.tag=="Tank") {
          collision.SendMessage("TakeDamege");
        }
      Destroy(this.gameObject);
    }

和简单，这个OnTriggerEnter()函数是内置API，传入一个碰撞器，当检测碰撞到tag为"Tank"时，就发送一个信号给这个物体，让其执行"TakeDamege"这个函数。这个“TakeDamege”是关于坦克血量的，稍后再说。

此时，只是很简单的碰撞，后面我们还希望碰撞之后有一个炮弹爆炸的特效。

如何显示坦克的血量

坦克的血量需要单独一个脚本来控制。在TankHp.cs这个文件里，建立一下函数。

 
x
    public float hp = 150;
    private float hpTotal;
    public Slider slider;
​
    void Start () {
        hpTotal = hp;
    }
    
​
    void TakeDamege(){
        if (hp <= 0) { return; }
        hp -= Random.Range(10,20);
        slider.value = hp / hpTotal;
        }
    }

先声明血量，设定血量为150。因为血条是固定的，因此血量需要以一定的比例显示在血条上。因此，还需要设置一个变量还接收总血量。当然，也可以直接设定总血量为150。在炮弹攻击到目标时，我们曾发送过一条信息给“TakeDamege”这个函数。现在，这个函数需要处理的，就是如何减少血量。

在TakeDamege()里，先判断血量是否为0，如果为0，则直接返回，不做任何处理，否则，随机在血量减少10,20之间的任意数值。

然后让slider的数值，等于目前血量除以总血量的比值。

在引擎方面，我们需要建立一个血条，这个是UI的设计了，不多讲。

如何控制摄像机跟随

摄像机的控制必须满足两个方面。第一，每一个坦克照到的范围应该一致。第二，无时无刻都要照到两个坦克

因此，必须满足

  • 摄像机始终在两个坦克中间
  • 两个坦克越分开，则摄像机的位置越高
 
x
    public Transform tank1, tank2;
    public Camera camera;
    private Vector3 offset;
    void Start () {
        camera = this.GetComponent<Camera>();
        offset = transform.position - (tank1.transform.position + tank2.transform.position) / 2;
    }
    void Update () {
        transform.position = offset + (tank1.position + tank2.position) / 2;
        float destance = Vector3.Distance(tank1.transform.position,tank2.transform.position);
        float size = destance * 0.58f;
        camera.orthographicSize = size;
    }

最先，要得到两个坦克的位置。其次，因为摄像机可以有多个，因此必须确定摄像机。

Start()里，获取摄像机。之后，offset等于摄像机的初始位置，就是摄像机的位置减去两个坦克位置之和的二分之一，即保持在两个坦克中间。

Update()函数里，每帧变更一次当前摄像机的位置。下一步，获取两个坦克的距离，Vector3.Distance()函数返回的是float的值。之后摄像机的size为距离的0.58倍，也就是，随着距离扩大，位置升高，最后告诉摄像机的Size位置就是size这个值。

如何分别控制两坦克

在“如何控制坦克的行动”里，有设置了编号这个变量。其承载着不同坦克的标志。当为1时，就采用1号方式控制坦克；为2，就采用第二号控制方案。控制方案在引擎的按键设置里更改。同时，不要忘记更改攻击的按键。

如何加特效

加背景音乐

因为摄像机会一直存在，所以将背景音乐这个对象放在摄像机上没有多大问题。直接添加组件，"Auido Source"就好了。

加音效

音效是在背景音乐上多了一些东西。首先，要了解音效和背景音乐有什么不同。背景音乐可能会一直播放，但是音效就是在触发了某个动作之后，才播放，动作结束的瞬间，音效结束。要控制那么多东西，肯定与脚本脱不了干系。

因为这个项目加入音效的地方挺多的，就只简单举一个例子来说明。

在坦克移动部分的脚本，TankMovement.cs里，加入以下的代码（记不住可以往回看一下。没有在函数体里的，就是在类里）

 
x
    public AudioClip idleAudio;
    public AudioClip drivingAudio;
    public AudioSource audio;
    ...
    void Start () {
        ...
        audio = this.GetComponent<AudioSource>();
    }
    
    void Update () {
        ...
        if (h != 0 || v != 0)
        {
            audio.clip = drivingAudio;
            if (audio.isPlaying == false)
            {
                audio.Play();
            }
        }
        else {
            audio.clip = idleAudio;

            }
        }
    }

AudioClip变量主要放入的是音效的片段，它本身不具有播放功能，需要AudioSource类型来播放。就类似碟片和唱片机的关系。于是，在Start()里，我们先获取AudioSource组件，下面的几次判断之后，都是将AudioClip放入到AudioSource这个唱片机里，如果另一个音效没有播放，则播放。
